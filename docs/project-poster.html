Pourquoi ce projet existe

Le hackathon ne récompense pas juste un joli texte. Il score chaque caption sur deux axes : fidélité à la vidéo et respect du style. Le but est donc de séparer compréhension factuelle et écriture créative.

Objectif produit

Transformer une vidéo courte en captions utilisables par des créateurs, médias ou outils d’accessibilité qui veulent adapter le ton sans réanalyser la vidéo.

Objectif concours

Maximiser Caption Accuracy et Style Match, tout en évitant les zéros automatiques : JSON cassé, style manquant, timeout ou image Docker inaccessible.

Stratégie gagnante

Comprendre la vidéo une seule fois, produire des faits neutres, puis générer les quatre styles en parallèle avec Gemma/Qwen/MiniMax via Fireworks.

Ce que j’ai fait

Chaque onglet explique une famille de changements et son intérêt concret pour le concours.

Robustesse : ne jamais perdre sur une erreur de forme

  • Validation Pydantic des entrées et sorties.
  • Fallbacks non vides si modèle, réseau ou timeout échoue.
  • Normalisation runtime alignée avec le self-check : longueur max, styles présents, jargon filtré.
  • Pas de secrets dans l’image ni dans le repo.

Score : améliorer accuracy + style match

  • Architecture deux étages : comprendre une fois, styliser quatre fois.
  • Prompts séparés pour formal, sarcastic, humorous_tech et humorous_non_tech.
  • Fallback modèles : Qwen2.5-VL 7B, Qwen2.5-VL 32B, MiniMax M3, Gemma 3 27B.
  • Audio optionnel via Whisper pour enrichir les faits quand disponible.

Soumission : éviter les zéros automatiques

  • Docker buildx en linux/amd64.
  • Run monté avec /input et /output.
  • Scripts de publication et vérification publique.
  • GitHub Actions pour preflight offline et build Docker.

Audit : méthode Ponytail

Ponytail n’a pas été ajouté comme dépendance. Je l’ai utilisé comme règle de décision : réutiliser l’existant, corriger le point central, ne pas ajouter de framework, garder les preuves. Résultat : fallback multi-modèles et streaming HTTP dans le pipeline, sans usine à gaz.

Pour passer du solide au compétitif

Les prochaines preuves nécessaires pour viser la première place, pas seulement une soumission valide.

1. Clé Fireworks

Lancer la vraie inférence VLM/style, comparer les captions et exécuter le juge miroir. C’est la preuve la plus importante pour le score.

2. Dataset 20+ clips

Tester nature, urbain, animaux, personnes, sport, food, weather et tech. Les 3 clips exemple ne suffisent pas pour conclure.

3. Image publique

Publier sur GHCR ou Docker Hub, vérifier un pull anonyme, la taille compressée, puis chronométrer un run complet réel.
